# Project Constitution

本文件是本项目最高技术规则。

所有架构设计、数据库设计、代码实现、AI生成代码，
必须遵守本文件。

若与其他规则冲突：
本文件优先级最高。

--------------------------------------------------

# 第一条：核心价值排序

优先级如下：

1. 安全
2. 稳定
3. 性能
4. 可维护性
5. 可读性
6. 优雅

--------------------------------------------------

# 第二条：架构原则

1. 严格分层架构
2. 禁止跨层调用
3. 依赖方向不可逆
4. 单一职责原则
5. 禁止巨型类（>300行）
6. 禁止巨型方法（>80行）

--------------------------------------------------

# 第三条：数据原则

1. 数据变更必须可追溯
2. 必须包含 create_time / update_time
3. 默认逻辑删除
4. 禁止 N+1 查询
5. 禁止无索引高频查询

--------------------------------------------------

# 第四条：事务原则

1. 事务只能存在于 Service 层
2. 查询方法禁止开启事务
3. 禁止大事务（>1000条更新）
4. 禁止跨服务事务

--------------------------------------------------

# 第五条：安全红线

严禁：

- SQL 拼接
- 信任前端参数
- 打印敏感信息
- 返回异常堆栈
- 硬编码密钥

--------------------------------------------------

# 第六条：变更原则

1. 数据库变更必须可回滚
2. 禁止直接线上改表
3. 重大架构变更必须评估影响

--------------------------------------------------

# 违反处理

违反本宪法的代码：

- 必须拒绝
- 必须重构
- 不允许以进度为借口绕过

--------------------------------------------------

# Final Statement

本文件具有最高权威。
AI 若生成违反条款的代码，必须拒绝输出。
